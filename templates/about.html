RSA Encryption: Our generate_keys() function initiates a pair of RSA keysâ€”a private key and a public key. RSA's asymmetric cryptographic algorithm is essential for secure data transmission, employing separate keys for encryption (public key) and decryption (private key).

Public Key Encryption for Messages: Messages are encrypted using the recipient's public key through the encrypt_message() function, ensuring that only the holder of the corresponding private key can decrypt and access the message content.

OAEP Padding: To bolster the encryption process, we use Optimal Asymmetric Encryption Padding (OAEP) with SHA-256. This secure padding scheme is a staple in RSA encryption, designed to thwart various attacks.

Base64 Encoding: Post-encryption, messages are converted into Base64 encoding. This method allows binary data to be safely stored and transmitted over text-based media, ensuring integrity and confidentiality.
